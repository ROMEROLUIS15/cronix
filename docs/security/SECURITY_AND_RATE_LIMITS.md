# 🛡️ Cronix: Security and Resource Consumption (Rate Limits)

This document details the **Defense in Depth** architecture implemented to protect the Cronix platform against abuse, brute-force attacks, and resource depletion.

---

## 📱 WhatsApp Defense Layers (AI Agent)

The WhatsApp processing flow features **7 layers of protection** before consuming expensive AI resources.

### 1. Integrity Verification (SHA-256)
*   **Purpose:** Validates that every incoming webhook truly originates from Meta.
*   **File:** `whatsapp-webhook/index.ts`

### 2. Resilience Queue (QStash)
*   **Purpose:** Message buffer to meet WhatsApp's 10s response time requirement and handle automatic retries.
*   **Retry budget:** `Upstash-Retries: 5` per message.
*   **Transparent rate-limit retry:** When Groq returns 429, `process-whatsapp` responds `503 + Retry-After: N` (where N comes from Groq's `retry-after` header). QStash waits exactly N seconds and retries automatically — the client never receives an error message.
*   **Deduplication:** `Upstash-Deduplication-Id: msg.id` prevents processing the same Meta event twice even under retry storms.

### 3. Circuit Breaker (AI Resilience)
*   **Limit:** 3 consecutive failures from Groq.
*   **Action:** If the AI fails, the system "opens the circuit" for 2 minutes to avoid saturating processes and provide user-friendly feedback.
*   **SQL:** `fn_wa_check_circuit_breaker`

### 4. User Rate Limit (Anti-Spam)
*   **Limit:** 10 messages / minute per phone number.
*   **SQL:** `fn_wa_check_rate_limit`

### 5. Business Rate Limit (Anti-Flood)
*   **Limit:** 50 messages / minute per tenant.
*   **SQL:** `fn_wa_check_business_limit`

### 6. Token Quota (Cost Control)
*   **Limit:** 50,000 tokens / day per business.
*   **Action:** Tracks real consumption ($) in Groq tokens. Once the limit is reached, the bot pauses until the next day.
*   **SQL:** `fn_wa_check_token_quota`

### 7. Media Filter (Infrastructure Protection)
*   **Limit:** 5.0 MB per audio/image file.
*   **Action:** Prevents downloading massive files from Meta's servers.

## 💻 Web & API Security

Protection injected via `middleware.ts` for the administrative platform.

### 1. Login Protection
*   **Limit:** 5 attempts / minute per IP.
*   **Protects:** Brute-force password attacks.

### 2. Global API Rate Limit
*   **Limit:** 60 requests / minute per IP.
*   **Protects:** Scraping and DDoS attacks on `/api/*` services.

## 📊 Monitoring and Maintenance

### Suspicious Activity View
To see which IPs or businesses are being blocked in real-time, run this in the SQL Editor:
```sql
SELECT * FROM v_web_suspicious_activity;
```

### How to increase limits
For a VIP business requiring more capacity:
1. Go to the `businesses` table.
2. Edit the `settings` (JSONB) field.
3. Add/Modify: `"wa_daily_token_limit": 500000`.
